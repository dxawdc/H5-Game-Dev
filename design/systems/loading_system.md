# Loading系统 系统策划案

**文档版本**：v1.0  **系统ID**：loading_system  **优先级**：P0  **复杂度**：S
**所属游戏**：围住Boss（Surround the Boss）
**对应脚本**：LoadingScreen.ts  **模块类型**：Scene

---

## 一、定位与目标

### 1.1 系统定位

Loading系统是游戏的入口画面，玩家启动游戏后见到的第一个界面。其核心职责是在游戏资源完成加载前向玩家展示品牌信息与合规内容，同时提供明确的加载进度反馈，消除等待焦虑。

### 1.2 目标

- 在资源未就绪时阻止玩家进入主界面，防止因资源缺失导致的渲染异常或崩溃。
- 展示游戏Logo，强化品牌认知。
- 展示"48字真言"（健康游戏忠告），满足国家新闻出版署对网络游戏的合规要求。
- 通过进度条实时反馈加载状态，告知玩家当前进度，减少等待焦虑。
- 通过缓存机制实现"秒开"体验：当游戏在短时间内再次启动时，跳过Loading直接进入主菜单。

### 1.3 用户场景

- **场景A（首次启动）**：玩家第一次打开"围住Boss"。本地无任何缓存，需要从CDN或微信小游戏包内加载所有资源（Logo图片、界面背景图、关卡配置、音效文件等）。Loading画面完整展示，进度条从0%逐步走到100%，然后自动跳转主菜单。
- **场景B（热启动/缓存命中）**：玩家在短时间内（默认30分钟内）再次启动游戏。上次加载的资源仍在微信小游戏缓存有效期内，跳过Loading画面直接进入主菜单。玩家感知为"秒开"。
- **场景C（缓存过期）**：超过30分钟未启动游戏，缓存被视为过期。重新展示完整Loading流程，但CDN资源可能命中HTTP缓存，加载速度仍快于首次启动。
- **场景D（资源加载失败）**：网络异常或CDN资源不可用导致加载中断。Loading画面停留在当前进度，显示重试按钮，玩家点击后重新尝试加载。

### 1.4 与其他系统的关系

| 系统 | 关系 |
|-----|------|
| 主界面系统 | Loading完成后自动跳转到主界面；加载前不渲染主界面 |
| GameData单例 | 读取GameData中的缓存时间戳（lastLaunchTimestamp）判断是否跳过Loading |
| 资源管理模块 | Loading系统作为资源加载的调用方，将异步加载进度映射为进度条视觉反馈 |

## 三、核心玩法规则

Loading系统不属于玩法系统，但包含以下核心规则：

### 3.1 启动判定规则

- 从 localStorage 读取键 `lastLaunchTimestamp`。
- 若不存在（首次启动）：走完整Loading流程。
- 若存在：计算 `当前时间戳 - lastLaunchTimestamp`。
  - 差值 < 1800 秒（30分钟）：判定为"热启动"，跳过Loading，直接切换到 MainMenu。
  - 差值 >= 1800 秒：判定为"缓存过期"，走完整Loading流程。
- 完整Loading流程开始后，立即将 `lastLaunchTimestamp` 更新为当前时间戳并写入 localStorage。

### 3.2 资源加载进度计算规则

总进度按资源数量加权平均，每种资源的权重由其预估大小决定：

| 资源类型 | 文件示例 | 预估大小 | 权重 |
|---------|---------|---------|------|
| Logo图片 | `assets/logo.png` | 80KB | 20 |
| 主菜单背景 | `assets/bg_main_menu.png` | 150KB | 30 |
| Boss角色精灵 | `assets/boss_default.png` | 60KB | 15 |
| 网格贴图 | `assets/grid_tile.png` | 40KB | 10 |
| 障碍物贴图 | `assets/obstacle.png` | 20KB | 5 |
| 道具图标（x4） | `assets/icon_*.png` | 16KB x 4 | 10 |
| 音效文件 | `assets/sfx_*.mp3` | 100KB | 10 |

合计权重 = 100。每完成一个资源的加载，`已完成权重 / 总权重` 即为当前进度百分比。进度值裁剪到 [0, 100] 闭区间，仅当 == 100 时标记加载完成。

### 3.3 进度条动画规则

- 进度条填充动画使用 CSS transition（Web）或 Canvas tween，动画时长固定 **300ms**（缓动函数 `ease-out`）。
- 当加载进度快速跳跃时，视觉进度条平滑追赶实际进度，避免画面闪烁。
- 进度条到达100%后，额外保留 **500ms** 的"完成展示时间"，让玩家看到完整的进度条后再切换场景。

### 3.4 Logo与48字真言展示规则

- Logo 居于画布垂直方向的 **35%~45%** 区域水平居中。
- Logo 下方间隔 **24px** 展示"48字真言"，文本配置如下：

```
抵制不良游戏，拒绝盗版游戏。
注意自我保护，谨防受骗上当。
适度游戏益脑，沉迷游戏伤身。
合理安排时间，享受健康生活。
```

- 48字真言字体大小 **14px**，颜色 `#999999`，行高 **22px**，每行两句（含标点16字符），共6行。

## 四、完整流程与状态机

### 4.1 状态机定义

```
                     ┌──────────────┐
         ┌──────────>│   IDLE       │
         │           └──────┬───────┘
         │                  │ 检测到启动
         │                  v
         │           ┌──────────────┐
         │           │ CHECKING     │ ← 读取 lastLaunchTimestamp
         │           └──┬─────┬─────┘
         │              │     │
         │        缓存有效  缓存无效/首次
         │              │     │
         │              v     v
         │           ┌──────────────┐     ┌──────────────────┐
         │           │ SKIPPED      │     │ LOADING          │ ← 加载资源，更新进度
         │           └──────┬───────┘     └────────┬─────────┘
         │                  │                      │
         │                  │ 切场景               │ 加载完成
         │                  │                      │
         │                  v                      v
         │           ┌─────────────────────────────────┐
         │           │ TRANSITIONING                    │ ← 淡出动画 500ms
         │           └──────────────┬───────────────────┘
         │                          │
         │                          v
         │           ┌─────────────────────────────────┐
         └───────────│       MAIN_MENU (外部场景)       │
                     └─────────────────────────────────┘
```

### 4.2 完整业务流程

1. **入口**：游戏启动，SceneManager 加载 LoadingScreen 场景实例。
2. **判定**（CHECKING 态）：从 localStorage 读取 `lastLaunchTimestamp`，执行 3.1 的启动判定逻辑。
3. **分支A - 跳过加载**（SKIPPED 态）：直接调用 SceneManager.switchTo('MainMenu')，流程结束。
4. **分支B - 完整加载**（LOADING 态）：
   - 更新 `lastLaunchTimestamp` 为当前时间戳。
   - 绘制Logo图片和48字真言。
   - 绘制进度条背景轨道（灰色）。
   - 启动资源加载队列，监听每个资源的 `onload` / `onerror` 事件。
   - 每完成一个资源，根据 3.2 的加权规则重新计算总进度，调用进度条更新。
   - 若加载过程无异常，进度达到 100%。
5. **过渡**（TRANSITIONING 态）：
   - 等待 500ms 完成展示时间。
   - 执行全屏淡出动画（透明度 1→0，时长 400ms，缓动 `ease-in`）。
   - 动画结束后调用 SceneManager.switchTo('MainMenu')。
6. **异常分支**（ERROR 态）：
   - 加载超时（超过 15 秒无任何资源完成）或资源加载出错（onerror 触发）。
   - 显示错误提示："资源加载失败，请检查网络连接"。
   - 显示"重试"按钮。
   - 玩家点击重试 → 回到 LOADING 态，重新初始化加载队列。

### 4.3 场景切换协议

```typescript
// 伪代码：场景切换器协议
interface ISceneTransition {
  from: string;      // 'Loading'
  to: string;        // 'MainMenu'
  animation: 'fade_out';
  duration: 400;     // ms
  onComplete: () => void;  // SceneManager.switchTo 调用入口
}
```

## 五、边界条件与异常处理

### 5.1 边界条件

| 编号 | 条件 | 预期行为 |
|-----|------|---------|
| B1 | 本地存储中 `lastLaunchTimestamp` 值为非数字 | 视为首次启动，走完整Loading。修复后覆写为正确时间戳 |
| B2 | 加载过程中玩家切到后台再切回 | 加载队列继续执行，不做暂停处理 |
| B3 | 进度条瞬间从0跳到100（极快加载） | 进度条仍然播放 300ms 动画平滑到达终点，不闪烁 |
| B4 | 加载总权重不足100（少数资源跳过） | 将完成权重重新归一化到 [0,100] 区间，避免进度条卡在 99% |
| B5 | 系统时间被用户篡改（回拨） | 差值计算可能为负，取绝对值处理：小于30分钟视为热启动 |

### 5.2 异常处理

| 编号 | 异常类型 | 处理策略 |
|-----|---------|---------|
| E1 | 单个资源加载失败（onerror） | 记录失败日志到 pipeline_errors.json（开发期），跳过该资源继续加载剩余资源。若该资源为非关键资源（如音效），降级使用静默失败；若为关键资源（Logo图片），向玩家显示错误提示 |
| E2 | 网络超时（15秒无响应） | 展示超时提示 + 重试按钮。重试时不清除已加载成功的资源缓存 |
| E3 | localStorage 不可用（微信小游戏低版本或隐私模式） | try/catch 捕获异常，跳过缓存判定逻辑，按"首次启动"处理 |
| E4 | 图片加载成功但尺寸异常（宽或高为0） | 视为加载失败，按 E1 处理 |

### 5.3 错误恢复

发生 E1-E4 任一异常后，系统进入 ERROR 状态。玩家点击重试按钮后的恢复流程：

1. 保留所有已成功加载的资源（不重复下载）。
2. 重新尝试所有失败/未完成的资源。
3. 若重试后全部成功 → 恢复正常流程，进入 TRANSITIONING。
4. 若重试后仍有失败 → 再次进入 ERROR 状态，最多允许重试 3 次。
5. 第 3 次重试仍失败 → 显示"加载失败，请重启游戏"，不再提供重试按钮。

## 六、前后端通信设计

### 6.1 通信概览

Loading系统属于纯客户端系统，无服务端实时通信。所有资源从微信小游戏的代码包（`wx.getFileSystemManager()`）或 CDN 静默资源加载，无需 HTTP API 调用。

### 6.2 事件总线协议

Loading系统通过全局事件总线（EventEmitter）向外派发以下事件，供后续系统（如主界面、埋点模块）监听：

| 事件名 | 方向 | 载荷 | 触发时机 |
|--------|------|------|---------|
| `loadStarted` | Loading → EventBus | `{ timestamp: number, cacheHit: boolean }` | 判定完成，开始加载资源时触发。`cacheHit` 为 `true` 表示跳过加载 |
| `loadProgress` | Loading → EventBus | `{ progress: number, loadedCount: number, totalCount: number }` | 每完成一个资源时触发。`progress` 为 0-100 的整数 |
| `loadCompleted` | Loading → EventBus | `{ totalTime: number, totalSize: number, cacheHit: boolean }` | 所有资源加载完成，进入 TRANSITIONING 前触发。`totalTime` 为加载耗时（毫秒），`totalSize` 为加载总字节数 |
| `loadSkipped` | Loading → EventBus | `{ reason: 'cache_hit' }` | 热启动判定通过，跳过加载时触发 |

事件编码规范：`camelCase`，动词过去式，符合 CLAUDE.md P2 约定。

## 七、美术需求说明

### 7.1 资源清单

| 资源ID | 资源名称 | 用途 | 格式 | 尺寸 | 数量 |
|--------|---------|------|------|------|------|
| `logo` | 游戏Logo | Loading界面主视觉，展示"围住Boss"游戏标题 | PNG（带透明通道） | 324x180px | 1 |
| `loading_bar_bg` | 进度条背景 | 空心轨道，浅灰色 | PNG | 432x16px | 1 |
| `loading_bar_fill` | 进度条填充 | 实心填充条，游戏主题色（橘色/红色系，与Boss色调一致） | PNG | 432x16px | 1 |
| `loading_bg` | Loading背景图 | 浅色渐变背景，含轻微装饰纹理 | JPG | 648x1152px | 1 |

### 7.2 视觉规范

- **配色**：进度条填充色使用 `#FF6B35`（活力橘色），与Boss红色系呼应。背景为 `#FFF8F0` 至 `#FFE8D0` 的垂直渐变。
- **Logo位置**：水平居中，垂直方向距顶部约 35%~45% 区域（即 y ≈ 403~518px，基于 1152px 高度）。
- **进度条位置**：Logo 下方间距 80px，水平居中，y ≈ 598~614px。
- **进度文字**：进度条正下方间距 12px，字号 13px，颜色 `#666666`，内容为"资源加载中......"（固定文本，不随进度变化）。
- **百分比数字**：进度条右侧间距 12px，字号 14px，颜色 `#FF6B35`，动态显示如 "65%"。
- **48字真言**：Logo 下方间距 24px，居中对齐，字号 14px，颜色 `#999999`，行高 22px。

### 7.3 动画规范

| 元素 | 动画 | 参数 |
|------|------|------|
| Logo | 淡入 | 时长 600ms，透明度 0→1，缓动 `ease-out` |
| 48字真言 | 淡入（延迟Logo 200ms后开始） | 时长 400ms，透明度 0→1 |
| 进度条填充 | 宽度过渡动画 | 时长 300ms，缓动 `ease-out` |

## 八、数据埋点需求

### 8.1 埋点事件列表

| 事件ID | 事件名 | 触发时机 | 携带参数 |
|--------|-------|---------|---------|
| `ld_start` | 加载开始 | 完整Loading流程启动 | `{ cacheHit: false }` |
| `ld_skip` | 加载跳过 | 热启动判定通过 | `{ reason: 'cache_hit', idle_seconds: number }`，`idle_seconds` 为距离上次启动的秒数 |
| `ld_progress` | 加载进度 | 每完成一个资源（最多每2秒上报一次去重） | `{ progress: number }` |
| `ld_complete` | 加载完成 | 资源全部加载成功 | `{ total_time_ms: number, total_size_bytes: number, resource_count: number }` |
| `ld_error` | 加载失败 | 资源加载出错 | `{ failed_resource: string, error_type: string, retry_count: number }` |
| `ld_retry` | 重试点击 | 玩家点击重试按钮 | `{ retry_count: number }` |
| `ld_retry_exhausted` | 重试耗尽 | 3次重试均失败 | `{ total_elapsed_ms: number }` |

### 8.2 性能指标关注

- **T0**（完全加载耗时）：从 `ld_start` 到 `ld_complete` 的毫秒数。目标值：首次启动 < 3 秒，缓存命中 < 0.5 秒。
- **T1**（可交互耗时）：玩家看到主菜单的耗时。目标值：≤ T0 + 1000ms（含过渡动画）。
- **加载成功率**：`ld_complete / (ld_start + ld_skip)`。目标值：≥ 99.5%。

## 九、迭代规划

### 9.1 P0（当前版本）

- 完整的启动判定逻辑（localStorage 缓存检测）。
- Logo + 48字真言展示，文案硬编码。
- 加权进度条 + "资源加载中......" + 百分比文字。
- 所有核心资源的加载队列管理。
- 加载完成后淡出动画 + 自动跳转主菜单。
- 基础异常处理（资源单个加载失败的重试）。

### 9.2 P1（近期优化）

- 48字真言文案通过外部配置文件管理，支持远程更新（响应政策要求）。
- 进度条加入微动效（填充时粒子效果或波纹渐变）。
- CDN 资源版本号管理（资源更新时避免缓存干扰）。
- 接入微信小游戏 `wx.getUpdateManager()`，实现静默更新下载并在下次启动时生效。

### 9.3 P2（远期规划）

- 加载页加入品牌动画（Logo 弹跳或 BOSS 角色出场动画），提升品质感。
- 背景图支持轮换（节假日/主题活动更换加载背景，如春节版红色背景）。
- 预加载策略优化：根据玩家历史行为预测下一场景，提前加载高频资源。
- 加载进度文本加入趣味 Tips（如"围住Boss的秘诀是..."），降低等待感。
