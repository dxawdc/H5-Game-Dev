# 设置系统 系统策划案

**文档版本**：v1.0  **系统ID**：settings_system  **优先级**：P1  **复杂度**：S
**所属游戏**：围住Boss（Surround the Boss）
**对应脚本**：SettingsSystem.ts  **模块类型**：Scene
**显示名**：设置（Settings）

---

## 一、定位与目标

### 1.1 系统定位

设置系统是"围住Boss"中提供玩家个性化配置和游戏内辅助操作的功能子系统。它以弹出层（Popup）形式呈现，既可从主界面齿轮图标进入，也可在关卡对局中点击设置按钮打开。系统承载两大核心职能：一是**偏好配置**——管理音效开关和震动开关，让玩家根据当前环境（如静音会议、深夜环境）灵活调整游戏感官反馈；二是**对局应急操作**——提供"退出挑战"功能，让玩家在陷入僵局或需要中断游戏时能干净地退出当前对局，并辅以退出确认对话框防止误触。

作为P1优先级系统，设置系统的音效和震动状态直接影响全局体验——音效开关控制着核心玩法中Boss移动音、放置障碍物音、胜利/失败音效等所有听觉反馈，震动开关控制触觉反馈。退出挑战功能则是对局状态机中一条关键的退出路径，需要与关卡系统的状态管理紧密协同。

### 1.2 核心目标

1. **个性化控制**：赋予玩家对游戏感官反馈的控制权，提升不同使用场景下的体验舒适度。例如在公共场所或夜间游玩时可关闭音效，在需要专注操作时可关闭震动。
2. **对局退出安全**：提供清晰的退出挑战流程，让玩家在关卡中途想离开时有明确的出口，同时通过确认对话框防止因误触导致的进度丢失。
3. **状态持久化**：音效和震动开关状态需跨对局、跨日持久化保存，玩家设定一次后无需重复调整。
4. **扩展预留**：为未来新增设置项（如画质选项、灵敏度调节、语言选择等）预留清晰的数据结构和UI扩展点。

### 1.3 与上下游系统的关系

| 系统ID | 关系类型 | 具体说明 |
|--------|---------|---------|
| `main_menu` | 上游 | 主界面齿轮图标按钮点击后打开设置弹出层；设置关闭后返回主界面 |
| `gameplay_core` | 上游 | 关卡内设置按钮点击后打开设置弹出层；退出挑战需通知核心玩法释放当前对局 |
| `level_system` | 依赖 | 退出挑战时需要将关卡状态重置，关卡系统提供重置接口 |
| `GameData` | 依赖 | 音效开关（`soundEnabled`）和震动开关（`vibrationEnabled`）数据存储在GameData单例中 |
| `analytics_system` | 输出 | 设置项变更和退出挑战操作上报埋点数据 |
| `audio_system` | 输出 | 音效开关变更时通知音频系统切换静音状态 |
| `scene_manager` | 依赖 | 退出挑战确认后，通过场景管理器切换回主界面场景 |

## 二、功能模块拆解

### 2.1 功能模块总览

| 编号 | 模块名称 | 类型 | 优先级 | 接入场景 | 依赖 |
|-----|---------|------|--------|---------|------|
| S1 | 音效开关 | 偏好配置 | P1 | 主界面设置 / 关卡内设置 | GameData, 音频系统 |
| S2 | 震动开关 | 偏好配置 | P1 | 主界面设置 / 关卡内设置 | GameData |
| S3 | 退出挑战 | 对局操作 | P1 | 仅关卡内设置 | 关卡系统, 场景管理器 |
| S4 | 退出确认对话框 | 交互确认 | P1 | 点击"退出挑战"后触发 | 无 |
| S5 | 版本信息展示 | 信息展示 | P1 | 主界面设置 / 关卡内设置 | 无 |
| S6 | 设置面板容器 | 容器框架 | P1 | 主界面设置 / 关卡内设置 | 场景管理器 |
| S7 | 未来设置扩展槽 | 预留扩展 | P2 | 主界面设置 / 关卡内设置 | 待定 |

### 2.2 模块详述

#### S1 音效开关

- **UI形态**：水平排列的标签"音效" + 开关滑块（Toggle Switch），开关滑块位于右侧
- **开关滑块规格**：宽度64px，高度32px，圆角16px；开启状态为绿色（#4CAF50），关闭状态为灰色（#BDBDBD）；滑块圆点直径28px，白色
- **默认状态**：开启（与GameData中 `soundEnabled` 的初始值 `true` 一致）
- **交互逻辑**：
  - 点击开关滑块切换状态，伴随200ms平滑过渡动画（滑块平移 + 背景色渐变）
  - 状态变更即刻写入 GameData.soundEnabled
  - 状态变更即刻同步到音频系统（`AudioSystem.setMuted(!soundEnabled)`）
  - 当前状态通过GameData持久化到localStorage，下次启动自动恢复
- **关联反馈**：切换音效开关时播放一次短暂的UI点击音效（若关闭音效则不播放），让玩家感知开关的实际效果

#### S2 震动开关

- **UI形态**：水平排列的标签"震动" + 开关滑块（Toggle Switch），布局与音效开关一致，位于音效开关下方
- **开关滑块规格**：同音效开关（64×32px，圆角16px）
- **默认状态**：开启（与GameData中 `vibrationEnabled` 的初始值 `true` 一致）
- **交互逻辑**：
  - 点击开关滑块切换状态，动画与音效开关一致
  - 状态变更即刻写入 GameData.vibrationEnabled
  - 状态变更即刻生效——震动开关关闭后，后续任何触控反馈（Boss移动震动、胜利/失败震动等）均不再触发
  - 当前状态通过GameData持久化到localStorage
- **微信小游戏适配**：震动功能调用微信小游戏API `wx.vibrateShort()`，震动开关关闭时跳过此API调用
- **关联反馈**：切换震动开关时，若开启则触发一次轻震动（`wx.vibrateShort({ type: 'light' })`），让玩家感知开关的实际效果

#### S3 退出挑战

- **UI形态**：红色文字按钮，居中展示，下方带分隔线与上方开关区域区分
- **文案**："退出挑战"
- **视觉样式**：文字颜色 #E53935（红色），字号28px，加粗；点击态颜色暗化20%
- **可见性条件**：仅当设置系统是从关卡内打开时（`context === 'in_level'`），才显示退出挑战按钮；从主界面打开时不显示
- **交互逻辑**：
  - 点击"退出挑战"按钮 → 关闭设置弹出层 → 打开退出确认对话框（S4模块）
  - 退出挑战操作不直接执行，必须经过确认对话框二次确认

#### S4 退出确认对话框

- **UI形态**：模态对话框，半透明遮罩 + 居中弹窗卡片
- **弹窗卡片规格**：宽度480px，高度约240px，白色背景圆角16px，带阴影
- **文案内容**：
  - 标题/内容文字："这次先放过Boss一马？"
  - 字体：字号32px，颜色 #333333，加粗，居中
  - 图标（可选）：Boss逃跑/挥手的卡通小图标，位于文字上方
- **按钮**：
  - "再想想"（取消按钮）：左侧，白底灰框，文字颜色 #666666，点击后关闭对话框，回到关卡对局
  - "放过它"（确认按钮）：右侧，红色渐变背景，文字白色，点击后执行退出挑战逻辑
  - 按钮尺寸：各180×64px，圆角8px，间距20px
- **交互逻辑**：
  - 点击遮罩区域：不关闭对话框（与普通弹出层不同，退出确认需强制玩家二选一）
  - "再想想"点击：关闭对话框，关卡对局继续保持，切回关卡界面
  - "放过它"点击：
    1. 关闭对话框
    2. 触发 `exitConfirmed` 事件
    3. 通知关卡系统重置当前对局状态（调用 `LevelManager.resetCurrentLevel()` 或类似接口）
    4. 清理关卡场景资源
    5. 场景切换到主界面（`MainMenu` 场景）
  - 确认退出后，当前对局的步数和进度不保存，不计入通关统计

#### S5 版本信息展示

- **UI形态**：小号灰色文字，位于设置面板底部
- **文案**："v{version}"（从 project_context 或 GameData 读取版本号）
- **字体**：字号16px，颜色 #999999
- **功能**：仅展示信息，不提供交互

#### S6 设置面板容器

- **UI形态**：圆角矩形卡片弹出层，从屏幕左侧滑入（从主界面进入时）或从屏幕底部滑入（从关卡内进入时），半透明黑色遮罩
- **尺寸**：宽度500px，高度根据内容自适应（约420px）
- **背景**：白色半透明（rgba 255,255,255,0.95）或磨砂玻璃效果
- **圆角**：16px
- **遮罩**：全屏半透明黑色（rgba 0,0,0,0.5），点击遮罩关闭设置面板
- **关闭按钮**：面板右上角 "✕" 按钮，直径44×44px
- **入场动画**：从左侧滑入（主界面场景）/ 从底部上滑（关卡场景），持续250ms ease-out
- **出场动画**：向左侧滑出 / 向下滑出，持续200ms ease-in
- **关闭规则**：点击遮罩或关闭按钮均关闭面板；面板关闭时触发 `settingsClosed` 事件

## 三、核心玩法规则（设置面板交互规则）

### 3.1 设置面板的双入口上下文

设置系统根据打开入口不同，呈现不同的UI组合：

| 入口 | 触发方式 | 面板动画 | 包含模块 | 退出挑战按钮 |
|------|---------|---------|---------|-------------|
| 主界面齿轮图标 | 主界面左侧中部齿轮按钮点击 | 从左侧滑入 | S1音效 + S2震动 + S5版本信息 + S7预留位 | 不显示 |
| 关卡内设置按钮 | 关卡界面右上角设置图标点击 | 从底部上滑 | S1音效 + S2震动 + S3退出挑战 + S4确认框 + S5版本信息 + S7预留位 | 显示 |

### 3.2 中断对局中的设置行为

在关卡对局中打开设置面板时，核心玩法应自动暂停（Pause），防止在设置操作期间Boss继续寻路移动或计时器继续运行：

- **暂停规则**：设置面板打开时，关卡核心玩法的游戏循环暂停（`requestAnimationFrame` 继续但更新逻辑跳过）；设置面板关闭时，恢复游戏循环
- **暂停表现**：关卡画面保留在设置面板背后（可见但不可交互），Boss动画定格
- **暂停例外**：设置面板打开期间，不计算关卡用时；恢复后继续计时

### 3.3 退出挑战的状态保证

退出挑战需要确保以下前置条件，避免数据不一致：

1. **关卡状态检查**：确认当前确实存在活跃对局（`currentLevel > 0`），不存在则忽略退出操作
2. **数据不保存**：退出后当前对局的步数、用时、道具使用等数据均不写入GameData的统计数据
3. **道具归还**：若玩家在本局使用了道具，已消耗的道具不退还
4. **场景清理**：关卡场景（GameLevel）的资源（网格数据、Boss对象、障碍物列表等）应完全释放，避免内存泄漏
5. **事件解绑**：关卡中注册的事件监听（如 `stepPlaced`, `bossMoved` 等）应在退出时全部解绑

## 四、完整流程与状态机

### 4.1 设置系统状态定义

| 状态 | 说明 |
|-----|------|
| `CLOSED` | 设置面板未打开，系统处于非活跃状态 |
| `OPENING` | 设置面板入场动画播放中 |
| `ACTIVE` | 设置面板完全打开，可交互状态 |
| `CONFIRMING` | 退出确认对话框打开中，等待用户选择 |
| `EXITING` | 退出挑战执行中，场景切换过渡 |
| `CLOSING` | 设置面板出场动画播放中 |

### 4.2 完整状态流转图

```
                         [主界面齿轮点击] / [关卡内设置按钮点击]
                                   │
                                   ▼
                               OPENING
                              (入场动画 250ms)
                                   │
                                   ▼
                               ACTIVE ────────────────────┐
                              ╱│╲                         │
                    用户操作  │ │  ┌── 点击遮罩/关闭按钮    │
                   ╱         │ │  ▼                       │
          点击音效开关  点击震动开关  CLOSING               │
               │            │    (出场动画 200ms)          │
               ▼            ▼        │                    │
          状态切换       状态切换     ▼                    │
          (即时生效)     (即时生效)  CLOSED                │
                                                          │
                   [关卡内设置] 点击"退出挑战"               │
                           │                              │
                           ▼                              │
                       CONFIRMING                         │
                      ╱          ╲                        │
                "再想想"        "放过它"                    │
                    │              │                       │
                    ▼              ▼                       │
                ACTIVE          EXITING                    │
                    │          (场景切换)                   │
                    │              │                       │
                    └──回到ACTIVE  ▼                       │
                              CLOSED ───── 返回主界面 ────┘
```

### 4.3 详细流程描述

#### 流程A：从主界面打开设置

1. 玩家点击主界面左侧齿轮图标
2. 触发 `settingsOpened` 事件，载荷 `{ context: 'main_menu' }`
3. 设置面板进入 `OPENING` 状态：遮罩渐入（0→0.5 opacity，200ms），面板从左侧滑入
4. 面板就位后进入 `ACTIVE` 状态，显示S1音效开关、S2震动开关、S5版本信息
5. 退出挑战按钮不显示
6. 玩家操作开关或点击关闭/遮罩
7. 进入 `CLOSING` 状态：遮罩渐出，面板滑出
8. 进入 `CLOSED` 状态，触发 `settingsClosed` 事件

#### 流程B：从关卡内打开设置

1. 玩家点击关卡界面右上角设置图标
2. 触发 `settingsOpened` 事件，载荷 `{ context: 'in_level' }`
3. 核心玩法暂停（游戏循环进入暂停态）
4. 设置面板进入 `OPENING` 状态：遮罩渐入，面板从底部上滑
5. 面板就位后进入 `ACTIVE` 状态，显示S1音效开关、S2震动开关、S3退出挑战按钮、S5版本信息
6. 玩家操作开关、或点击退出挑战、或点击关闭/遮罩
7. **若关闭面板**：核心玩法恢复 → 面板出场动画 → `CLOSED`
8. **若点击退出挑战**：进入流程C

#### 流程C：退出挑战（含确认对话框）

1. 在 `ACTIVE` 状态下，玩家点击"退出挑战"按钮
2. 系统进入 `CONFIRMING` 状态：关闭设置面板 → 打开退出确认对话框
3. 对话框显示："这次先放过Boss一马？" + "再想想" / "放过它" 两个按钮
4. **玩家点击"再想想"**：
   - 关闭对话框
   - 回到 `ACTIVE` 状态（保留设置面板打开状态）
   - 或者直接回到关卡界面（关闭设置面板）
   - 推荐行为：回到 `ACTIVE` 状态，让玩家继续设置操作或关闭面板回关卡
5. **玩家点击"放过它"**：
   - 触发 `exitConfirmed` 事件，载荷 `{ levelId: number, elapsedTime: number, stepCount: number }`
   - 进入 `EXITING` 状态
   - 调用关卡系统接口重置当前对局（重置状态、清理资源）
   - 清理场景事件绑定
   - 场景切换至主界面（MainMenu）
   - 进入 `CLOSED` 状态

### 4.4 开关操作数据流

```
用户点击音效开关滑块
  → 滑块播放动画（0.2s 平移+变色）
  → 读取当前 GameData.soundEnabled
  → 取反：newValue = !currentValue
  → 写入 GameData.soundEnabled = newValue
  → 触发 soundToggled 事件，载荷 { enabled: newValue }
  → 通知 AudioSystem.setMuted(!newValue)
  → GameData.saveToStorage() 持久化
  → 结束
```

## 五、边界条件与异常处理

### 5.1 网络相关

| 场景 | 预期行为 |
|-----|---------|
| 无网络时打开设置 | 完全正常，设置系统不依赖网络 |
| 无网络时退出挑战 | 完全正常，退出挑战是本地操作 |
| 微信震动API调用失败 | `wx.vibrateShort()` 可能在低端机或特定微信版本不支持，使用 `try/catch` 包裹，失败时静默跳过，不影响开关状态切换 |

### 5.2 数据异常

| 场景 | 预期行为 |
|-----|---------|
| `soundEnabled` 在localStorage中缺失 | 读取时默认值为 `true`，首次保存时写入 `true` |
| `vibrationEnabled` 在localStorage中缺失 | 读取时默认值为 `true`，首次保存时写入 `true` |
| localStorage数据损坏 | GameData捕获解析异常，设置项恢复默认值（音效开、震动开），静默修复 |
| 设置项数据格式异常（非bool值） | 类型守卫检测，非bool时重置为默认值 `true`，写入报错日志 |

### 5.3 UI/交互异常

| 场景 | 预期行为 |
|-----|---------|
| 快速连点开关滑块 | 开关动画进行期间禁用点击，动画完成后恢复；保证状态最终与滑块显示一致 |
| 关卡内打开设置后，在设置界面中关卡状态变化 | 关卡已暂停，不会发生变化 |
| 退出挑战确认对话框打开时，切到微信后台 | 返回游戏时对话框仍保持打开状态 |
| 在设置打开时关卡结算触发 | 先关闭设置面板，再展示结算界面；或设置打开时禁止结算触发（推荐） |
| 退出挑战执行中（`EXITING`）再次点击退出按钮 | 按钮在 `EXITING` 状态下禁用，防止重复触发 |
| 极窄屏幕（<375px逻辑宽度） | 设置面板宽度自适应为屏幕宽度的90%，最大500px |
| 异形屏/刘海屏 | 设置面板顶部/底部避开安全区40px |

### 5.4 并发与时序异常

| 场景 | 预期行为 |
|-----|---------|
| 音效开关和震动开关同时快速切换 | 各自独立处理，互不阻塞，最终状态均为最后一次操作的结果 |
| 退出挑战清理过程中内存回收延迟 | 场景切换完成后执行一次垃圾回收提示（`wx.triggerGC()` 在微信小游戏中可用时调用） |
| 正在播放音效时关闭音效开关 | 当前正在播放的音效立即停止；后续音效不再播放，直至音效开关重新开启 |

## 六、前后端通信设计

### 6.1 通信方式

纯前端架构，无后端服务器。所有设置数据通过 GameData 单例管理，经 localStorage 持久化。设置面板通过全局事件总线（EventEmitter）与上下游系统通信。

| 数据类型 | 存储方式 | 读写时机 |
|---------|---------|---------|
| 音效开关状态 | GameData.soundEnabled → localStorage | 初始化时读取；开关切换时写入 |
| 震动开关状态 | GameData.vibrationEnabled → localStorage | 初始化时读取；开关切换时写入 |
| 设置面板打开上下文 | 运行时内存（无需持久化） | 打开时传入 |
| 游戏版本号 | project_context 读取 | 初始化时读取 |

### 6.2 事件总线协议

设置系统通过全局事件总线与上下游系统通信，所有事件采用 camelCase 命名：

| 事件名 | 方向 | 载荷 | 触发时机 |
|--------|------|------|---------|
| `settingsOpened` | 设置系统 → 全局 | `{ context: 'main_menu' | 'in_level' }` | 设置面板入场动画开始前 |
| `settingsClosed` | 设置系统 → 全局 | `{ soundEnabled: boolean, vibrationEnabled: boolean, context: string }` | 设置面板出场动画结束后 |
| `soundToggled` | 设置系统 → 音频系统 | `{ enabled: boolean }` | 玩家切换音效开关时 |
| `vibrationToggled` | 设置系统 → 全局 | `{ enabled: boolean }` | 玩家切换震动开关时 |
| `exitConfirmed` | 设置系统 → 关卡系统 | `{ levelId: number, elapsedTime: number, stepCount: number }` | 玩家在退出确认对话框中点击"放过它"时 |
| `gameplayPaused` | 关卡系统 → 设置系统 | `{}` | 关卡内打开设置时，关卡暂停 |
| `gameplayResumed` | 关卡系统 → 设置系统 | `{}` | 关卡内关闭设置时，关卡恢复 |
| `sceneTransitionStarted` | 设置系统 → 场景管理器 | `{ from: 'GameLevel', to: 'MainMenu' }` | 退出挑战确认后，场景切换前 |

## 八、数据埋点需求

### 8.1 埋点总表

| 事件ID | 事件名 | 触发时机 | 上报字段 | 用途 |
|--------|-------|---------|---------|------|
| `settings_open` | 打开设置面板 | 设置面板打开时 | `{context: 'main_menu'|'in_level'}` | 设置面板使用率，按入口统计 |
| `settings_close` | 关闭设置面板 | 设置面板关闭时 | `{context: string, duration_ms: number, close_type: 'button'|'mask'}` | 设置面板停留时长、关闭方式偏好 |
| `settings_sound_toggle` | 切换音效开关 | 音效开关状态变化时 | `{enabled: bool}` | 音效开关开启率、切换频次 |
| `settings_vibration_toggle` | 切换震动开关 | 震动开关状态变化时 | `{enabled: bool}` | 震动开关开启率、切换频次 |
| `settings_exit_click` | 点击退出挑战 | 点击退出挑战按钮时 | `{level_id: number, step_count: number}` | 退出挑战点击率 |
| `settings_exit_confirm` | 确认退出挑战 | 点击"放过它"按钮时 | `{level_id: number, step_count: number, elapsed_time: number}` | 实际退出率、退出时游戏进度分布 |
| `settings_exit_cancel` | 取消退出挑战 | 点击"再想想"按钮时 | `{level_id: number}` | 退出反悔率、"再想想"按钮有效性评估 |

### 8.2 埋点上报方式

- 使用微信小游戏 `wx.reportEvent` 上报或自定义HTTP上报
- 非关键路径事件延迟上报，可合并批量（每30秒或退出游戏时统一上报）
- 上报失败静默处理，不影响设置交互体验
- 设置项切换事件（sound_toggle, vibration_toggle）可做防抖处理，频繁切换时只上报最终状态

### 8.3 核心指标

| 指标 | 计算方式 | 目标值（参考） |
|-----|---------|--------------|
| 设置面板打开率（主界面） | 主界面设置打开次数 / 主界面PV | >15% |
| 设置面板打开率（关卡内） | 关卡内设置打开次数 / 对局开始次数 | >20% |
| 音效关闭率 | 音效关闭用户数 / 总用户数 | <30%（预期大部分用户保持开启） |
| 震动关闭率 | 震动关闭用户数 / 总用户数 | 30-50%（公共场所使用场景多） |
| 退出挑战率 | 退出挑战次数 / 关卡内设置打开次数 | <40%（退出是较重的操作） |
| 退出反悔率 | "再想想"点击次数 / 退出挑战点击次数 | >15%（过低说明确认框形同虚设） |

### 9.2 功能优先级依据

- **P1（v1.1基础体验）**：音效和震动开关影响核心游戏体验，是玩家最基础的个性化需求；退出挑战是对局状态机必要组成部分，缺失则玩家只能靠切后台或重开游戏来中断对局，体验极差。版本信息展示是合规基本要求。
- **P2（v2.0+扩展功能）**：画质选项、灵敏度调节等属于锦上添花的增强功能，主要影响特定场景下的体验优化和低端机适配，不阻塞核心流程。语言选择需配合全局多语言架构，投入较大。

### 9.3 技术债务与优化空间

| 项目 | 说明 | 计划版本 |
|-----|------|---------|
| 面板动画性能优化 | 使用CSS transform（或Canvas变换矩阵）实现动画，避免重绘 | v1.2.0 |
| 设置数据迁移方案 | 当设置项数据结构变更时（如新增字段），提供向后兼容的迁移逻辑 | v1.2.0 |
| 设置面板组件化 | 将开关滑块、按钮、分隔线等抽象为可复用UI组件，供其他系统（如排行榜弹出层）使用 | v1.2.0 |
| 微信震动API兼容检测 | 检测当前微信版本是否支持 `wx.vibrateShort`，不支持时自动隐藏震动开关 | v2.0.0 |

### 9.4 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|------|------|---------|
| 微信震动API在部分机型不可用 | 中 | 低 | try/catch 包裹，失败静默跳过；开关仍可正常切换，只是震动反馈不生效 |
| 玩家误触退出挑战导致投诉 | 低 | 高 | 二次确认对话框 + "再想想"默认聚焦，防止误触及 |
| 设置数据localStorage被清除 | 中 | 低 | 默认值兜底（音效开、震动开），不影响游戏正常运行 |
| 关卡内打开设置时对局状态未正确暂停 | 低 | 高 | 严格测试设置面板打开/关闭与关卡暂停/恢复的配对逻辑 |
